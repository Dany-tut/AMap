# AMaps

Карты, которые сами замечают, где ты побывал. Катаешься на байке, гуляешь или
едешь — приложение открывает «туман войны» над пройденными улицами, засчитывает
новые локации, предлагает прикрепить фото и заметку, выдаёт ачивки за
исследование города.

Механика Pokémon GO, но про реальные прогулки, а не про монстров.

## Почему это работает

Классические карты не следят, где ты бываешь, поэтому люди забывают
задокументировать места. AMaps переворачивает это: карта сама фиксирует визит и
просит его описать. Туман войны и коллекция «открытого города» создают
lock-in — жалко бросать наполовину раскрытую карту.

## Архитектура

Проект собран как Swift Package с чёткими слоями. Каждый слой не знает про
вышестоящие — домен и движки тестируются без iOS API.

| Модуль | Ответственность |
|---|---|
| `AMapsDomain` | Модели (сессии, ячейки, места, ачивки) и правила ачивок. Ноль зависимостей. |
| `AMapsFog` | Движок тумана войны: сетка (H3/slippy), множество открытых ячеек, конфликт-фри merge для синка, % открытости города. |
| `AMapsTracking` | Приём и фильтрация GPS-фиксов, накопление статистики сессии. Без CoreLocation. |
| `AMapsStorage` | Граница персистентности (`Store`). Прод — SQLite/GRDB, для тестов — `InMemoryStore`. |
| `AMapsApp` | Composition root: связывает слои. В Xcode оборачивается в SwiftUI `App`. |

### Ключевые решения

- **Туман = множество битов, не дырки в картинке.** Посещённая ячейка —
  один индекс в таблице. Merge между устройствами — union, идеальный CRDT →
  синк через CloudKit тривиален и приватен (маршруты лежат в iCloud юзера).
- **Сетка абстрагирована** (`GridProjection`). По умолчанию slippy-тайлы z17;
  прод подменяет на H3 res 10–11 без изменений в `FogEngine`.
- **Домен и движки — чистый Swift**, поэтому покрыты обычными юнит-тестами.

Полная схема, стек и план работ — в обсуждении проекта.

## Стек (прод)

SwiftUI · MapLibre Native + векторные тайлы (Protomaps/OSM) · GRDB/SQLite ·
CloudKit private DB · StoreKit 2 · CoreLocation (Visits + significant changes +
active session) · CoreMotion (автодетект активности).

## Сборка

```bash
swift build
swift test
```

Для iOS-приложения слои подключаются в Xcode-таргет как локальный пакет.

## Статус

Реализовано и покрыто тестами (8/8):
- модели и data-driven ачивки
- fog engine + обратная проекция ячеек в полигоны
- `FogGeoJSON` — генерация GeoJSON открытых зон для MapLibre (с фильтром по вьюпорту)
- `SessionRecorder` — фильтрация GPS-фиксов и статистика
- `LocationTracker` — мост к CoreLocation (passive/active режимы)
- `CloudKitSync` — конфликт-фри синк тумана через приватную iCloud-базу

Дальше — Xcode-таргет с SwiftUI-экранами и MapLibre-рендером, StoreKit-подписка,
замена slippy-сетки на H3.
